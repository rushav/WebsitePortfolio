Turtlebot3 Human Interaction

A custom ROS2 application for real-time person detection, autonomous human-following, and an interactive tag game — built on Turtlebot3 with YOLO and a custom fabricated enclosure.

In Progress ROS2 YOLO OpenCV Python LiDAR SolidWorks Fabrication
01 / Overview

What It Is

This project extends the Turtlebot3 platform with a custom ROS2 application combining real-time computer vision, state machine control, and interactive game logic. The robot can detect and follow a specific person through a crowd using YOLO-based detection, then transition into an interactive tag game where the robot attempts to "tag" participants by approaching them.

The project is being developed in the context of the UW GIX Labs Autonomous Robotics Studio curriculum. Beyond the software stack, a custom shell enclosure was designed in SolidWorks and fabricated — providing integrated mounting for the RGB-D camera, improved cable routing, and ventilation for the onboard compute.

The tag game logic uses a finite state machine with four states: idle/waiting, searching for target, following a detected person, and executing a "tag" approach. LiDAR-based collision avoidance runs in parallel to prevent obstacles from interfering with gameplay.

02 / What's Been Built

Completed Components

□
YOLO Person Detection
Real-time YOLOv8 person detection pipeline running on onboard compute. Publishes bounding box centroids to ROS2 topics for the control node to consume. Tested on live camera feed with sub-200ms detection latency.
▬
State Machine Controller
ROS2 node implementing a finite state machine with well-defined transitions: idle → searching → following → tagging. State transitions triggered by detection confidence, distance thresholds, and game events.
■
Custom Shell Enclosure
SolidWorks-designed enclosure fabricated via 3D printing. Integrates a camera mount at optimal detection height, ventilation slots for the Raspberry Pi compute, and cable management channels. Mounts directly to the Turtlebot3 base.
▭
LiDAR Collision Avoidance
360° LiDAR scan processed in real time to maintain a minimum safe distance from obstacles. Runs as a parallel ROS2 node and publishes safety override commands that supersede game state navigation commands when an obstruction is detected.
▫
RViz Simulation
Full RViz visualization stack for validating the perception and navigation pipeline before physical deployment. Costmap layers, detection overlays, and state indicators visible in real time for debugging and tuning.
▩
Human Following
Proportional controller on detected bounding box horizontal offset and area for smooth human-following behavior. Maintains a target following distance and adjusts heading to keep the person centered in frame.
03 / What's Coming

Upcoming Work

Development is ongoing. The following areas are planned for the next phases of this project:

Planned Additions

  • Sensor Refinement — Improved depth fusion between LiDAR and RGB-D camera for more robust 3D person localization and distance estimation in cluttered environments
  • Person Re-identification — Track a specific target across occlusions and crowd scenarios; explore appearance-based re-id to maintain target lock when the person temporarily leaves frame
  • Additional Game Modes — Expand beyond single-target tag to multiplayer scenarios, freeze-tag variants, and timed challenge modes with scoring
  • Field Testing — Move from corridor/lab testing to open-area field trials with multiple participants to stress-test detection, following, and collision avoidance under real conditions
  • Sound & LED Feedback — Integrate onboard buzzer and RGB LED ring to signal game states (searching, following, tagged) for a more engaging user experience
  • Enclosure Revision — Second-iteration enclosure based on field testing feedback — addressing thermal management, cable routing improvements, and aesthetic refinement
