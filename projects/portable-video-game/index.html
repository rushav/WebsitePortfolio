01 / Overview

What It Is

This is a fully self-contained portable video game built from scratch — hardware design, firmware, and physical enclosure included. The game is a vertical scrolling space shooter running on the ESP32-C3 microcontroller, displayed on a 128x64 OLED. The player controls a spaceship dodging and shooting obstacles, progressing through difficulty levels and competing for a persistent high score.

The physical design is what makes this project distinctive: the enclosure was designed in Fusion 360 and 3D printed to look like a miniature desktop PC workstation. The OLED display serves as the "monitor," the three game buttons are arranged in a keyboard-style layout, and the main ESP32 and electronics are housed in a tower enclosure sitting on a desk base. The rotary encoder is positioned on the base for menu navigation.

The entire game logic — collision detection, scoring, difficulty scaling, level progression, high score storage, menu system, startup animations — runs in CircuitPython across a modular set of source files.

The enclosure design was created in Fusion 360 (public model available at a360.co/3KH3QhY). The design uses three distinct physical sections that mirror the logical structure of the hardware:

  • Monitor: OLED display mounted at viewing angle on a raised stand
  • Keyboard area: Three game buttons arranged in an ergonomic left-center-right layout
  • Tower enclosure: Houses the ESP32-C3, power switch, NeoPixel, buzzer, MPU-6050, and all wiring
  • Desk base: Structural base that ties all elements together; rotary encoder is surface-mounted here
03 / Software & Game Logic

CircuitPython Architecture

The firmware is organized into modular CircuitPython source files, each with a single clear responsibility. This mirrors the physical design's component separation and makes the codebase easy to extend.

Core Modules
  • code.py — Main game loop, state management, collision detection, scoring
  • hardware.py — All hardware initialization and utility functions
  • game_objects.py — Obstacle and Bullet classes with self-managed position and collision
  • graphics.py — All drawing functions: grid background, HUD, ship, targeting reticle
  • rotary_encoder.py — Quadrature state machine with position wrapping
  • accelerometer_filter.py — EMA-filtered tap detection with auto-calibration
UI & Audio Modules
  • animations.py — Startup animations: Windows-style splash, animated radar HUD
  • menu.py — Main menu, difficulty selection, high score display, initials entry
  • sounds.py — All audio: menu nav, weapon fire, movement, game over, victory fanfare

Level Progression

Every 100 points: screen inverts colors, LED flashes, player must tap the device within 5 seconds to continue. Victory at 300 points: rainbow LED cycle, victory fanfare. High scores (top 5 with initials) saved to device filesystem across power cycles.

04 / Build Process

From CAD to Physical Device

The build process began with defining the component placement requirements — display viewing angle, button ergonomics, encoder placement — and translating those into the Fusion 360 enclosure model. After 3D printing and assembly, firmware was written iteratively with hardware connected at each stage.

The accelerometer calibration routine runs automatically on startup, establishing a gravity-compensated baseline using an Exponential Moving Average filter (alpha = 0.3) to reliably distinguish intentional taps from handling noise.

05 / Final Result

The Finished Device

The finished device is a compact, self-contained arcade game that fits on a desk. The miniature desktop PC aesthetic is immediately recognizable — the monitor (OLED), keyboard (buttons), and tower (ESP32 enclosure) create a cohesive and intuitive form factor. The rotary encoder sits naturally on the desk base for menu navigation.

The NeoPixel LED provides color-coded gameplay feedback visible through the tower enclosure: white for normal play, blue flash for lane changes, green for weapon fired, red for game over, and a rainbow cycle for victory.
